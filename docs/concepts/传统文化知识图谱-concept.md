## 01｜项目定位

- **项目名称**：传统文化知识图谱（落在随心而行 / Mystic Lab）
- **一句话定位**：不是算命工具，而是「占问过程中理解符号与推演逻辑」的学习型体验。
- **产品愿景**：把塔罗、小六壬、六爻（及未来梅花、八字等）拆成可学、可查、可对照的知识宇宙。
- **核心价值**：体验层给结果；学习层给原因；图鉴层给长期内容资产与留存。
- **目标用户**：对神秘学/传统文化好奇，想通过轻量仪式获得自我理解，又不满足于一句「吉凶」的年轻人。
- **使用场景**：临事想问一眼；事后对照应验；想弄懂「这个卦/六神/牌」为什么这么说。

## 02｜为什么做（Why）

**用户痛点**  
当前用户：

- 遇到什么问题？得到一句断语后不知道依据，学不进去，也留不住。
- 现在有什么解决方式？百科文章、短视频、传统书籍——与「正在问的问题」脱节。
- 为什么现有方案不好？工具型 App 偏娱乐；教材偏系统但难进入；两者中间缺「带着问题学一层」。

**产品机会**  
随心而行已有塔罗/小六壬体验与旅程；若只做结果，容易变成一次性玩具。知识图谱把「每一次占问」变成图鉴与学习进度的入口。

**核心理念**

> 六爻（及同类体系）**不是预测未来**。  
> 而是：**通过卦象（符号）帮助用户理解当前局势，找到行动方向。**

小六壬同理：时间趋势提醒，不是宿命宣判。

## 03｜产品架构（Information Architecture）

```
Mystic Lab 首页
├ 占问体系（塔罗 / 小六壬 / 梅花…）
├ 学习体系（入门路径 · 分层课程）
├ 我的旅程（历史 · 对照 · 笔记）
├ 图鉴（牌 / 六神 / 卦…）
└ 社区/分享（后置）
```

| 模块 | 用户目的 | 核心功能 | 页面 | 后续扩展 |
|---|---|---|---|---|
| 占问体系 | 完成一次有仪式感的提问 | 提问、仪式/起课、结果 | `/tarot` `/xiaoliuren` `/meihua` | 六爻起卦 |
| 学习体系 | 按层级学会「为什么」 | 路径、关卡、单符号课 | 学习首页 / 课页 | 六爻十层链路 |
| 我的旅程 | 回看与对照 | 聚合记录、应验、同题 | `/records` | 跨体系时间线 |
| 图鉴 | 查符号、收藏、深读 | 列表、深页四段/五层 | 塔罗图鉴 / 六神图鉴 / 卦象图鉴 | 统一 Schema |
| 社区/分享 | 表达与传播 | 心意卡、分享图 | 结果页导出 | 社区（P3） |

## 04｜用户体验流程（User Flow）

```
用户进入 Lab
↓ 提出问题
↓ 选择体系与方式
↓ 完成仪式 / 起课
↓ 生成结果
↓ 理解原因（分层解释）
↓ 学习知识（点进符号深页）
↓ 保存记录（旅程 / 应验）
```

| 步骤 | 用户行为 | 系统反馈 | 情绪目标 |
|---|---|---|---|
| 进入 | 选体系或看建议 | 短定位备注（心理探索/时间趋势/象与变化） | 不慌、知道选哪个 |
| 提问 | 输入问题 | 教练/改问（可选） | 问题变可答 |
| 仪式 | 手势或起课步骤 | 过程可见、可学 | 仪式感、参与感 |
| 结果 | 读摘要 | 先一句，再分层展开 | 「有依据」 |
| 理解 | 展开第 2–5 层 | 组成、关系、行动 | 懂一点，不是被算 |
| 学习 | 进图鉴深页 | 故事 + 映射 + 组成 | 想收藏、想再来 |
| 保存 | 写感悟 / 标应验 | 旅程聚合、三天对照 | 长期关系 |

## 05｜核心体验设计

### 功能 A｜分层结果解释（所有体系共用模板）

| 字段 | 内容 |
|---|---|
| 功能名称 | 五层解释（可裁剪） |
| 用户为什么需要 | 拒绝「只给吉凶」 |
| 交互方式 | 默认见第 1 层；展开 2–5 |
| 页面 | 结果页 |
| 动效 | 层递展开 |
| 数据 | 符号 id + 问题类型 + 快照 |
| AI 作用 | 第 4–5 层结合问题生成；第 1–3 层优先结构化知识 |

### 功能 B｜六爻起卦体验（样板；可后置实现）

用户：「最近工作机会怎么样？」

```
选择：铜钱起卦 / 时间起卦 / 随机起卦
↓
生成：本卦 · 变卦 · 动爻
↓
进入解释（五层）
```

### 功能 C｜单卦深页（例：泽火革）

不是只写「革卦：改变。」结束，而是：

1. **认识这个卦**  
   - 上卦兑 ☱：泽 · 交流、喜悦、开放  
   - 下卦离 ☲：火 · 热情、行动、变化  
   - 组合：泽上火下，火烧于泽下 → 旧状态被推动 → **革**＝改变、更新、突破旧模式  
2. **卦象故事**：火欲上、泽欲容，相遇而生变（好记）  
3. **生活映射**：事业转型 / 关系调模式 / 成长放旧习  
4. **学习组成**：兑+离 → 兑金离火 → 火克金 → 为何成「革」

## 06｜知识体系设计

路径：

```
入门理解 → 基础规则 → 组成结构 → 案例学习 → 个人记录 → 进阶探索
```

六爻学习链路（每层可成模块）：

```
起卦 → 八卦 → 六十四卦 → 六爻结构 → 阴阳变化
→ 六亲 → 世应 → 动爻 → 变卦 → 结合问题解释
```

结果五层：

1. 这个卦是什么？  
2. 为什么形成这个卦？  
3. 每个爻代表什么？  
4. 和你的问题有什么关系？  
5. 你可以怎么行动？

**试点建议**：先用「小六壬六神图鉴」或「塔罗单牌深页」跑通同一模板，再铺六十四卦。

## 07｜页面规划

### Page 01｜Hero（Lab / 体系首页）

- **目的**：第一眼理解「占问 + 可学」  
- **视觉**：夜空仪式感 + 体系入口  
- **交互**：三体系卡片 + 体系建议小字  
- **素材**：emblem / 掌图预览  
- **动画**：轻入场，不挡点击  

### Page 02｜核心体验（起课 / 抽牌）

- **标题**：随心速问 / 随心占问  
- **文案**：先看眼前这一步；答案在你心里  
- **布局**：提问 → 过程 → 结果分层  
- **组件**：问题框、仪式区、结果层、进图鉴 CTA  

### Page 03｜符号深页（图鉴）

- **布局**：一句 → 展开详解 → 故事 / 映射 / 组成  
- **组件**：关键词、分层章节、收藏与笔记  

### Page 04｜我的旅程

- **目的**：对照与留存  
- **组件**：分区、同题对照、应验标记  

## 08｜视觉设计

- **风格关键词**：东方古籍档案 · 现代知识库 · 仪式感  
- **色彩**：深底 + 金/纸色强调；六爻可用青绿墨韵区分塔罗紫金  
- **字体**：标题有书法气质但不牺牲屏读；正文清晰无衬线  
- **材质**：纸、印、轻微纤维；避免廉价霓虹  
- **动效**：翻页/展开/落位，短而稳  
- **组件**：图鉴卡、分层解释、时间线旅程条  

## 09｜技术方案

- **前端**：现有 Vite + TypeScript SPA（Mystic Lab）  
- **数据**：本机 localStorage；知识条目 Schema 单源（如图鉴 JSON/TS）  
- **AI**：可选 LLM 做第 4–5 层与问法教练；第 1–3 层以结构化知识为准  
- **动画**：CSS + 现有仪式动效；复杂叙事页可评估 GSAP  

## 10｜版本规划

- **P0**：统一「一句 + 分层展开」模板；小六壬/塔罗各跑通 1 条深页  
- **P1**：内容完善（六神全文、更多牌）；旅程对照巩固  
- **P2**：学习路径（关卡/进度）；知识图谱导航  
- **P3**：社区/分享  
- **P4**：AI 增强（按问题生成学习路径、对照复盘）  
- **六爻起卦完整流**：放在 P1 后或独立里程碑，避免拖垮 P0  

## 11｜未来扩展

```
六爻 → 梅花 → 奇门 → 八字 → 紫微 → 知识宇宙
```

与现有：塔罗（心理探索）· 小六壬（时间趋势）· 梅花（象与变化）并列，共用「体验 + 学习 + 图鉴」骨架。

## 12｜产品洞察

**这个项目真正有价值的地方是什么？**

不是功能清单本身，而是：

> **把复杂传统知识系统，重新设计成现代用户可以理解和参与的体验。**

用户带走的不只是「这一次怎么说」，还有「我开始懂一层结构」——这才是留存与作品集深度。
