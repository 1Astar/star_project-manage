## 2. 项目背景（Why / Context）

### 分析

- **为什么做**：单纯「抽牌看结果」容易撞车、用完即走；「学习 + 图鉴 + 手札」才有长期价值与作品集叙事深度。
- **原始灵感**：塔罗手势抽牌 — 用摄像头识别手势，让抽牌过程有仪式感，而非点按钮。
- **用户痛点**：得到一句断语后不知道依据；百科/短视频与「正在问的问题」脱节；传统 App 偏娱乐，教材偏系统难进入。
- **市场/竞品问题**：工具型产品强调「准不准」；内容型产品强调「学体系」；中间缺「带着问题学一层、事后能对照」。
- **核心理念**：

> 占问不是预测未来，而是通过符号帮助理解当前局势，找到行动方向。  
> 小六壬是时间趋势提醒，不是宿命宣判。

### Why / Context 页输出

| 项 | 内容 |
|---|---|
| **页面标题** | Why · 占问之后，还想知道「为什么」 |
| **核心文案** | 年轻人不缺占卜入口，缺的是：**一次有仪式感的提问 + 可回看的理解轨迹**。随心而行把塔罗、小六壬放进同一 Lab，让每次占问成为图鉴与旅程的入口。 |
| **视觉表达** | 深色星空底 + 三体系 emblem 并列；Lab 首页像「神秘学档案室」入口，而非娱乐 App 宫格。 |
| **交互方式** | 首屏轻动效星层；卡片 hover 微抬；三体系短定位以括号小注呈现（心理探索 / 时间趋势 / 象与变化）。 |

## 3. 产品目标

| 维度 | 内容 |
|---|---|
| **用户问题** | 临事想问一眼；事后想对照应验；想弄懂「这个牌/六神为什么这么说」；同一问题想换体系交叉验证。 |
| **产品目标** | ① 完成有仪式感的占问闭环 ② 结果可理解、可收藏、可回看 ③ 跨体系统一「我的旅程」 ④ 为后续知识图谱/学习路径留接口 |
| **设计原则** | **仪式先于效率** — 手势/掌诀/时辰盘降低「随便点一下」的廉价感。**理解先于断语** — 结果页分层：传统含义 → 结合问题 → 行动提醒。**对照先于预测** — 手札记录 + 3 天后待对照，强调复盘而非宿命。**同题可切换** — 塔罗与小六壬服务不同问法，但同一问题可互跳一眼。 |

## 5. 核心体验设计

### 体验 A · 手势塔罗抽牌

```
用户：带着具体问题进入抽牌页
↓ 操作：对着摄像头做选手势，左右挥浏览牌堆，确认后抽取
↓ 系统反馈：牌面翻转动画 + 牌义 Tab（传统/结合问题/行动/图鉴）
↓ 结果：可存手札、进图鉴看「第一次相遇」、反馈有呼应/不准
```

**为什么这样设计**：手势是仪式载体，区别于「随机数生成器」类竞品。  
**解决什么**：抽牌过程的参与感与记忆点。  
**取舍**：依赖 HTTPS 与摄像头权限；微信内置浏览器提供触控降级，不强行全平台手势。

### 体验 B · 小六壬掌上起课 + 待对照

```
用户：问「这件事最近顺不顺」
↓ 操作：看时辰盘/掌诀教学 → 在掌图顺数起课 → 得六神
↓ 系统反馈：传统含义 / 结合问题 / 现在更适合 / 提醒（五段结构）
↓ 结果：写入手札；满 3 天首页/旅程置顶「待对照」→ 标记应验/未应验
```

**为什么这样设计**：小六壬强项是「短期趋势」，对照闭环强化「趋势提醒」而非「预言准确」。  
**解决什么**：占问用完即走；用户无法验证解读是否有助于行动。  
**取舍**：浏览器通知仅做「回站试用」，不做后台准时推送（控制范围）。

### 体验 C · 同题跨体系互跳

```
用户：塔罗问完「要不要换工作」
↓ 操作：结果页点「也用小六壬看一眼」
↓ 系统反馈：sessionStorage 预填同题，跳转小六壬提问页
↓ 结果：旅程详情展示两体系摘要对照
```

**为什么这样设计**：不同体系服务不同问法，但用户常对同一困惑多角度看。  
**解决什么**：体系孤岛、重复输入问题。  
**取舍**：不做自动双开解读，保持用户主动选择。

## 9. Reflection

**学到了什么**

- 文化类产品要先定「不是什么」（算命工具），再定体验结构；否则功能堆叠会失焦。
- 手势/仪式是差异化，但**对照闭环**才是留存与作品集叙事的关键证据。
- 多体系并存时，**统一旅程 + 同题互跳**比再做第四个体系更重要。

**最重要的设计决策**

1. 定位从「手势塔罗」升级为「随心而行 / Mystic Lab」
2. 结果页信息架构：传统 → 结合问题 → 行动（而非单一 AI 段落）
3. 手札「待对照」— 把占问从一次性娱乐变成可复盘档案
4. AI 问法教练「按需调用」— 避免边输入边调 API 的干扰与成本

**如果继续，会怎么发展**

- **P1**：知识库深页（符号五层解释），与 `idea-040a2f8f` 知识图谱 Idea 对齐
- **P2**：旅程子页（收藏/进度/笔记）与分享心意卡
- **P3**：梅花易数「象与变化」主流程（先定文案与问法，不急做功能）
- **不做**：关页后准时推送、完整六爻起卦（资源有限时后置）
