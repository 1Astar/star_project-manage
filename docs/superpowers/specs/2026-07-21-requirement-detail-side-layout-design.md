# 需求详情页左右布局

日期：2026-07-21  
范围：仅 `/projects/[id]/requirements/[reqId]`

## 目标

一屏内看完主要信息并完成测试/验收/讨论；对齐禅道式左右栏，避免正文在上、协作在下的长页。

## 布局

- 大屏：`grid` 左约 2/3、右约 1/3；小屏上下叠
- 左：标题与正文编辑（描述、验收、场景等）
- 右（自上而下、压紧）：基本信息摘要 → 角色任务 → 测试核对 → 验收项 → 讨论
- 保留测试核对「通过 / 不通过」

## 不做

- 不改需求池、项目概况、设置
- 不做底部浮动工具栏、不做历史时间线（除非已有数据源）
